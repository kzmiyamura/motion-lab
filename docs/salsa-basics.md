## 2. On1 vs On2 比較

| 特徴 | On1（LA スタイル） | On2（NY / マンボスタイル） |
|------|-------------------|--------------------------|
| ブレイクステップのタイミング | **1拍目**（強拍）で前進 | **2拍目**（コンガのスラップ音）でステップバック |
| リズムの取り方 | 明快な強拍に合わせるため取りやすい | メロディの揺らぎ（ビヨーン感）を表現しやすい |
| 見た目・スタイル | 力強く、派手、ショー的 | 優雅、洗練、アーバン |
| 難易度 | 初心者向き | 中・上級者向き（音楽の解釈が必要） |
| 主な場所（日本） | 多くのソーシャル全般 | 東京・大阪などの主要都市イベント |

**推奨学習順序**: On1 → 慣れたら On2

### On1 ステップタイミング（リード / 男性視点）
- **1**: 左足を前（ブレイク）
- **2**: 右足を元の位置
- **3**: 左足を元の位置
- **4**: 休止
- **5**: 右足を後ろ（ブレイク）
- **6**: 左足を元の位置
- **7**: 右足を元の位置
- **8**: 休止

### On2 ステップタイミング（リード / 男性視点）
- **1**: 左足をその場で踏む（待つ）
- **2**: 右足を後ろ（ブレイク）
- **3**: 左足を元の位置
- **4**: 休止
- **5**: 右足をその場で踏む
- **6**: 左足を前（ブレイク）
- **7**: 右足を元の位置
- **8**: 休止

## 3. ビートカウントの基本

- サルサは **8ビート構造** を使用
- 8ビートがある音楽であればジャンル問わず踊れる（ポップ・ロックはテンポがやや遅め）

### ステップを踏むカウント

```
1  2  3  [4休]  5  6  7  [8休]
↑  ↑  ↑          ↑  ↑  ↑
Step Step Step  Step Step Step
```

- **踏む**: 1, 2, 3 / 5, 6, 7
- **休止（ポーズ）**: 4, 8

## 4. ベーシックステップ

男女が向かい合って踊るため、リード（男性）が前進するときフォロー（女性）は後進する（衝突防止のミラー構造）。

### リード（男性）のベーシックステップ

| カウント | 動作 | 足 |
|---------|------|----|
| 1 | 前に1歩出す | 右足 |
| 2 | 元の位置で足踏み | 左足 |
| 3 | 元の位置に戻す | 右足 |
| 4 | **休止** | - |
| 5 | 後ろに1歩下げる | 左足 |
| 6 | 元の位置で足踏み | 右足 |
| 7 | 元の位置に戻す | 左足 |
| 8 | **休止** | - |

### フォロー（女性）のベーシックステップ

| カウント | 動作 | 足 |
|---------|------|----|
| 1 | 後ろに1歩下げる | 左足 |
| 2 | 元の位置で足踏み | 右足 |
| 3 | 元の位置に戻す | 左足 |
| 4 | **休止** | - |
| 5 | 前に1歩出す | 右足 |
| 6 | 元の位置で足踏み | 左足 |
| 7 | 元の位置に戻す | 右足 |
| 8 | **休止** | - |

### ベーシックのバリエーション

5カウント目の動き方はリードが決める：

| バリエーション | リード（男性）5カウント | フォロー（女性）5カウント |
|--------------|----------------------|------------------------|
| 標準 | 左足を**後ろ**に | 右足を**前**に |
| バリエーション | 左足を**前**に | 右足を**後ろ**に |

## 5. 基本ステップ一覧

### サイドステップ

- ベーシックステップの前後移動を **左右移動** に変えたもの
- 開始足：リードは右足、フォローは左足

| カウント | 動作 |
|---------|------|
| 1 | 横に1歩出す（右方向） |
| 2, 3 | 足を戻して左右交互に足踏み |
| 4 | **休止** |
| 5 | 反対側に1歩横にステップ（左方向） |
| 6, 7 | 同様に足踏み |
| 8 | **休止** |

---

### クロスステップ

- サイドステップに似た構造
- **3カウント目と7カウント目** で反対の足を前からクロスさせてステップを踏む

---

### スージーQ（Suzie Q）

- クロスステップと同様に左右の足をクロスさせる動き
- 繰り返すと元の位置に戻らず **少しずつ前進** していく軌跡が特徴

---

### マンボ（Mambo）

- **例外的なステップ**：通常の 1,2,3 / 5,6,7 のカウント構造が崩れる
- 1,2 と 3,4 を **同じ足** で踏む必要があるため、体重移動を伴わない
- **重心はその場に置いたまま、足だけを動かす** のがポイント

> ⚠️ 骨格解析での検出ヒント：重心（腰の中点）がほぼ静止しているのに足首だけが動いている状態

---

### アラウンド・ザ・ワールド（Around the World）

- 最初に立っていた位置を中心として、ステップを踏みながら **その場でぐるっと一周** する動き

---

## 6. 骨格解析への応用メモ（開発用）

| ステップ | 検出のヒント |
|---------|------------|
| ベーシック | 足首の前後移動（Z軸 / Y軸）。1拍目または2拍目のブレイクを基準に周期性を見る |
| サイドステップ | 足首の X 軸方向への最大振幅で判別 |
| クロスステップ | 膝（25, 26）と足首（27, 28）の X 座標が **交差** する状態を検出 |
| スージーQ | クロス状態が繰り返しながら腰中点が前進するパターン |
| マンボ | 腰中点（23, 24 の中点）の移動距離が小さいのに足首が動いている状態 |
| アラウンド・ザ・ワールド | 腰中点が円弧を描いて一周する軌跡 |

### Beat-phase と動作の対応（On1）

| カウント | 期待される動き |
|---------|--------------|
| 1, 5 | ブレイクステップ（最大変位） |
| 2, 3 / 6, 7 | 戻り・足踏み（速い小さな移動） |
| 4, 8 | **休止**（足速度 ≈ 0）← Z-filtering で微細ブレはノイズ除去 |

## 7. ペアワーク（ルーティン）技術仕様

### 7-1. ペアワーク概要

- リード（男性）とフォロー（女性）が音楽に合わせて **ルーティン（連続技）** を踊る
- 基本姿勢・体重移動・リード＆フォローの感覚が核心
- 代表的な技：ライトターン / ハンマーロック / ドロップ＆キャッチ など

---

### 7-2. 技術カテゴリ別 物理解析仕様表

| カテゴリ | 技名 | 物理的特徴・トリガー | 判定に使う Landmark |
|---------|------|---------------------|-------------------|
| 基本移動 (Foundation) | ベーシック・サイド・クロス | 前後・左右の足の位相ズレ（1-2-3 / 5-6-7） | 27（左足首）、28（右足首）の Z/X 座標 |
| 位置交換 (Cross Body) | クロスボディ・リード (CBL) | リーダーが道を開け、フォロワーが 180° 入れ替わる | 23, 24（腰）の XY 平面上での 180° 旋回 |
| 単体回転 (Turns) | ライトターン / 左ターン | 頭部を軸とした 360° 高速回転。軸の垂直維持が鍵 | 0（鼻）、7, 8（耳）の水平回転速度 |
| 腕の拘束 (Arm Work) | ハンマーロック (Hammerlock) | 片腕が背中に回り、肘が鋭角に固定 | 11-13-15（左腕）または 12-14-16（右腕）の角度判定 |
| リリース (Release) | ドロップ＆キャッチ (Drop & Catch) | 手の接触が一瞬離れ、再度捕捉 | 15, 16（手首）の距離急増 → 急接近 |
| 回り込み (Around) | アラウンド・ザ・ワールド | リーダーの周囲をフォロワーが円軌道で公転 | 腰（23, 24）の重心移動が円軌道を描く |
| 首/肩の技 (Neck/Shoulder) | ネックロール / ショルダーチェック | 腕が相手の首や肩を経由、高低差が大きい | 15, 16（手首）が 0（鼻）や 11, 12（肩）の高さを通過 |
| 足技 (Footwork) | スージーQ / シャイン | 足首が交互に交差し、重心が細かく移動 | 27, 28 の X 座標の頻繁な反転（交差） |
| 静止/演出 (Dips) | ディップ / ポーズ | 重心が垂直軸から大きく外れ、低重心でホールド | 全身 Velocity ≈ 0、かつ 0（鼻）が 23（腰）より低い |

---

### 7-3. 技の自動識別（Pattern Recognition）

手首（15, 16）と肩・腰の相対距離、および足首（27, 28）の交差パターンから、
現在の動きが以下のどれに近いかを**確率で表示**する：

- `CBL`（クロスボディ・リード）
- `RightTurn`（ライトターン）
- `Hammerlock`（ハンマーロック）

**判定ロジックの方針：**

| 技 | 主な判定条件 |
|----|------------|
| CBL | 腰（23, 24）が 180° 旋回 + 前後への大きな XY 変位 |
| ライトターン | 鼻（0）・耳（7, 8）の水平回転速度が高い + 腰の垂直軸が維持されている |
| ハンマーロック | 片腕（例：12-14-16）の肘角度が鋭角、かつ手首が腰後方に位置する |

---

### 7-4. コネクション・ロスト検知

**ドロップ＆キャッチ** などで手が離れる瞬間を検知し、その後のキャッチ成功可否をログに残す：

1. **ドロップ検知**: 手首（15, 16）間の距離が急増（閾値：正規化座標で 0.2 以上の急増）
2. **キャッチ成功**: ドロップ検知後 N フレーム以内に手首距離が再び接近（閾値内に戻る）
3. **キャッチ失敗 / タイムアウト**: N フレーム経過しても接近しない場合はミスとして記録

---

### 7-5. 物理的エラー検知（上達のヒント）

#### ターンの軸ブレ（ライトターン）

- **条件**: ターン中に鼻（0）が腰中点から水平（X 軸）に **15% 以上ズレ**た場合
- **表示**: 警告色（赤 / オレンジ）でオーバーレイ描画

```
axisError = abs(nose.x - hipMid.x) / renderW > 0.15
```

#### ハンマーロックの危険性

- **条件**: 肘（13 または 14）の角度が鋭角になりすぎた場合
- **表示**: 肘の関節を警告色で強調表示し、注意を促す

```
elbowAngle = angle(shoulder → elbow → wrist)
if elbowAngle < 45° → WARNING
```

### 7-6. マルチポーズ・ターゲット分離（ペアワーク専用）

ペアワーク中は2人が密着・重なり合うため、通常の Nearest Neighbor では ID が混同する。

**リード / フォロー 分離のロジック：**

| 判定方法 | 詳細 |
|---------|------|
| 物理ベクトル区別 | ベーシックステップの1カウント目、**前進する人物 = リード、後進する人物 = フォロー** として初期 ID を割り当てる |
| 重なり解消後の維持 | オクルージョン中は速度ベクトル（vx, vy）で予測位置を追跡し、解消後も正確に各自の ID を維持 |
| サイズ比 補助判定 | 身長・肩幅の比率が大きく異なる場合は補助的な手がかりとして使用（体型差がある場合のみ有効） |

> **注意**: 体型が近い2人の分離は現状の Lite モデルでは困難。Full モデル + ワールド座標の利用を検討。

## 8. インストラクター・チェックポイント × AI解析仕様（5レイヤー）

世界中のサルサインストラクターが見るチェックポイントを、AI骨格解析ロジックに対応させた仕様表。

### L1: 軸（Axis）

| チェックポイント | インストラクターが見ること | AI 判定ロジック |
|----------------|--------------------------|----------------|
| **スポッティング** | 回転中に頭が最後に残り、最初に戻るか。首のムチのような動き | 0（鼻）、7, 8（耳）の **角速度の急激な変化（スパイク）** を検出 |
| **センター・オブ・グラビティ** | 体重移動時に骨盤が浮かず、常に水平を保っているか | 23, 24（腰）の **Y 座標の変動幅**（上下動の抑制）を監視 |

---

### L2: 接続（Connection）

| チェックポイント | インストラクターが見ること | AI 判定ロジック |
|----------------|--------------------------|----------------|
| **テンション＆プレッシャー** | 腕が伸び切らず、肘が常に適度な遊び（スプリング）を持っているか | 11-13-15（腕）の角度が **90°〜160° の範囲内**か |
| **フレーム（The Frame）** | 腕だけでリードせず、肩ラインと腕の三角形が維持されているか | 11, 12（肩）と 15, 16（手首）が形成する **三角形の面積** を維持しているか |

---

### L3: 足捌き（Footwork）

| チェックポイント | インストラクターが見ること | AI 判定ロジック |
|----------------|--------------------------|----------------|
| **インサイド・エッジ** | 着地時に足裏全体ではなく、親指側に体重が乗っているか | 27, 28（足首）と 31, 32（足先）の **接地角度の微差** |
| **トランスファー** | 3カウント目で完全に体重が乗り切り、軸足が入れ替わっているか | 重心（23, 24 の中点）が **支持脚の真上に移動**したか |

---

### L4: 技（Patterns）

| チェックポイント | インストラクターが見ること | AI 判定ロジック |
|----------------|--------------------------|----------------|
| **クロスボディ・リード** | リーダーが 2ビート目で道を空け（Open the door）、フォロワーの進路を確保したか | 11, 12（肩）のベクトルが **90° 回転**し、フォロワーの Z 軸移動を許容しているか |
| **コパ（Copa / In & Out）** | フォロワーを一度引き寄せ（In）、反動で外へ送り出す（Out）加速度があるか | 15, 16（手首）の **急減速（1〜3カウント）→ 急加速（5〜7カウント）** への転換 |

---

### L5: 音楽性（Musicality）

| チェックポイント | インストラクターが見ること | AI 判定ロジック |
|----------------|--------------------------|----------------|
| **オン・ワン（On 1）** | 1ビート目の強拍と足の着地が 0.01秒単位で一致しているか | 音声波形のピークと 27, 28（足首）の **速度ゼロ（着地）** の同期 |
| **ブレイク（Break Step）** | 前後の入れ替え時、一瞬の「タメ」があるか | 足首の移動ベクトルが **反転する瞬間の静止時間** を計測 |

---

### 5レイヤー全体のまとめ

```
L1 軸     → 頭・腰の垂直安定性（鼻・耳・腰 の座標）
L2 接続   → 腕の角度・三角形フレームの維持（肩・肘・手首）
L3 足捌き → 重心移動と着地タイミング（足首・足先・腰中点）
L4 技     → パターン認識（肩ベクトル回転・手首加速度）
L5 音楽性 → ビートとの同期精度（足首着地 × 音声波形）
```
